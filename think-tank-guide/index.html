00
준비물 설치
Claude Code와 Obsidian 설치하기

1. Obsidian 설치

obsidian.md에서 무료 다운로드 후 설치하세요. 설치 후 볼트(Vault)를 하나 만들어두세요. 위치는 어디든 상관없어요.

2. Node.js 설치

Claude Code를 설치하려면 Node.js가 필요해요.

확인 방법 
# 터미널에서 아래 입력 — 버전 숫자가 나오면 이미 설치됨
node --version
💡
버전 숫자가 안 나오면 nodejs.org에서 설치하세요. LTS 버전 선택.

3. Claude Code 설치

터미널 
npm install -g @anthropic-ai/claude-code
💡
터미널이 뭔가요? 맥에서 Command + Space → "터미널" 검색 → 실행. 검은 화면에 명령어를 입력하는 곳이에요.
01
think-tank 폴더 만들기
Claude Code의 작업 공간 생성

터미널을 열고 아래 명령어를 한 줄씩 입력하세요.

터미널 — 한 줄씩 입력 
# 1. 홈 폴더에 think-tank 폴더 생성
mkdir ~/think-tank

# 2. 그 폴더로 이동
cd ~/think-tank

# 3. Claude Code 실행
claude
💡
~/think-tank 가 뭔가요?
~는 내 컴퓨터의 홈 폴더를 뜻해요.
즉 /Users/내이름/think-tank와 같아요.
⚠️
Obsidian 볼트 안에 만들 필요 없어요. 완전히 별개의 폴더예요.
think-tank = 필터 엔진  /  Obsidian = 저장소
02
Claude Code 신뢰 설정
처음 실행 시 1회만 필요

Claude Code가 처음 실행되면 아래 화면이 나와요.

화면 출력
Quick safety check: Is this a project you created or one you trust?

  2. No, exit

1 입력 후 Enter를 누르세요. 이후엔 이 화면이 다시 나오지 않아요.

03
핵심 파일 생성
Claude Code에게 "나는 이런 사람이야"를 알려주는 파일들

Claude Code 입력창에 아래 내용을 전체 복사해서 붙여넣기하세요.
[ ] 안의 내용만 본인 정보로 수정하면 돼요.

Claude Code에 붙여넣기 
아래 구조로 파일들을 만들어줘.

1. CLAUDE.md 파일:
---
# About Me
[본인 소개 — 예: 나는 UX 디자이너이며 Make.com 자동화를 공부하고 있다]
비개발자이며, 노코드 자동화(Make.com)를 중심으로 공부하고 있다.

# 지식을 쌓는 목적 (우선순위 순)
1. [목적 1]
2. [목적 2]
3. [목적 3]

# 행동 규칙
- 정보를 받으면 반드시 4단계 검증을 거친다
- 검증 없이 수용 금지
- 인사이트는 항상 내 목적 중 어디에 쓰이는가로 연결할 것
- 저장 경로는 Obsidian 볼트 기준으로 안내할 것

# Obsidian 볼트 경로
[본인 Obsidian 볼트 경로 — 예: /Users/username/Documents/Obsidian Vault]
---

2. CONTEXT.md 파일:
---
# 현재 진행 중인 것들
- Think Tank 시스템 구축 중
- [현재 공부 중인 것들 추가]
---

3. memory/ 폴더 생성
4. inbox/ 폴더 생성
5. sessions/ 폴더 생성

6. .claude/commands/intake/index.md 파일:
---
description: "기사, 글, 텍스트를 붙여넣을 때 자동 실행"
---

# 지식 인테이크 검증 프로토콜

사용자가 기사, 글, 텍스트를 붙여넣으면 아래 4단계를 자동 실행한다.

## 4단계 검증
① 출처 — 1차 자료인가, 누군가의 해석인가?
② 근거 — 데이터/사실인가, 의견/추측인가?
③ 반대 시나리오 — 이게 틀릴 수 있는 조건은?
④ 적용성 — 내 목적 중 어디에 쓰이는가?

## 저장 안내
검증 후 Obsidian의 어느 폴더에 저장할지 추천한다.
---

완료되면 아래처럼 체크 표시가 나와요:

완료 화면
think-tank/
├── CLAUDE.md          ✅
├── CONTEXT.md         ✅
├── memory/            ✅
├── inbox/             ✅
├── sessions/          ✅
└── .claude/
    └── commands/
        └── intake/
            └── index.md  ✅
💡
각 파일의 역할:
CLAUDE.md — 매 대화 시작 시 자동으로 읽힘. 내가 누구인지 기억
CONTEXT.md — 지금 뭘 하고 있는지 맥락 유지
memory/ — 대화가 길어져도 사라지지 않는 장기 기억
inbox/ — PDF 파일을 여기 넣으면 자동 분석
.claude/commands/intake/index.md — 기사 붙여넣기 시 자동 검증 실행
04
Obsidian 폴더 구조 만들기
검증된 지식이 저장될 폴더들

Obsidian 앱을 열고 볼트 안에 아래 폴더들을 만드세요.
기존 폴더가 있으면 새로 만들 필요 없어요 — 비슷한 기존 폴더를 활용하면 돼요.

목적추천 폴더 이름저장할 내용
📥 임시 투하Inbox/처음엔 다 여기에, 나중에 분류
🤖 자동화 공부References/AI-automation/Make.com, AI 관련 인사이트
💼 업무/UXReferences/UX/팀 운영, 디자인 관련
💰 수익 아이디어Ideas/Business/자동화 수익 아이디어
🎨 개인 브랜드Ideas/Brand/캐릭터, 콘텐츠 관련
⚠️
폴더 이름은 완전히 자유예요. 위 이름이 싫으면 본인이 쓰던 이름으로 만들어도 돼요. 다음 단계에서 경로를 알려주면 그걸 기준으로 작동해요.
05
Obsidian 경로 연결하기
CLAUDE.md에 저장 위치 등록

Claude Code에 아래를 입력하세요. 폴더 이름은 Step 04에서 만든 것으로 수정해요.

Claude Code에 입력 
CLAUDE.md의 Obsidian 저장 경로를 아래로 업데이트해줘:

- 📥 임시 투하 → Inbox/
- 🤖 자동화 공부 → References/AI-automation/
- 💼 업무/UX → References/UX/
- 💰 수익 아이디어 → Ideas/Business/
- 🎨 브랜드 → Ideas/Brand/

그 다음 Claude Code가 ".claude/commands/intake/index.md도 동일하게 맞춰줄까요?" 라고 물으면 yes 입력하세요.

06
첫 번째 테스트
기사 하나 던져보기

세팅이 끝났어요. 실제로 작동하는지 확인해볼게요.

  1. 인터넷에서 관심 있는 기사를 찾는다 (자동화, UX, 무엇이든)
  2. 기사 텍스트를 전체 선택해서 복사한다
  3. Claude Code 입력창에 그냥 붙여넣기한다
  4. 자동으로 4단계 검증이 실행되는 걸 본다
  5. "References/AI-automation/ 에 저장하세요" 같은 안내를 확인한다
  6. Obsidian을 열어서 해당 폴더에 노트를 만들고 내용을 저장한다
⚠️
자동 검증이 실행 안 된다면 Claude Code에 /intake 를 먼저 입력한 뒤 기사를 붙여넣어 보세요.
✅
검증 후 저장 경로 안내가 나오면 시스템이 정상 작동하는 거예요!

자주 묻는 질문

think-tank 폴더를 Obsidian 안에 만들어야 하나요?
아니요. 완전히 별개예요. think-tank는 필터 엔진, Obsidian은 저장소예요. 섞이면 Claude Code 시스템 파일과 노트가 뒤섞여서 지저분해져요.
둘이 자동으로 연결되나요?
지금은 수동이에요. Claude Code가 "어디에 저장하세요"라고 알려주면 내가 직접 Obsidian에 붙여넣어요. Make.com으로 자동화도 가능해요.
Claude Code를 매번 새로 실행해야 하나요?
네. 쓸 때마다 터미널에서 cd ~/think-tank → claude 입력하면 돼요. 이전 맥락은 CLAUDE.md와 memory/ 폴더 덕분에 유지돼요.
CLAUDE.md는 언제 수정하면 되나요?
목적이 바뀌거나, 새로운 Obsidian 폴더를 추가했을 때 Claude Code에 "CLAUDE.md 업데이트해줘"라고 말하면 돼요.
유료 플랜이 필요한가요?
Claude Code는 Anthropic 계정이 필요해요. claude.ai에서 계정 생성 후 Claude Code를 실행하면 로그인하라는 안내가 나와요.
