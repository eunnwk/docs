Section 01

이 파이프라인이 하는 일

네이버 블로그는 JavaScript로 렌더링되는 Next.js 구조라 단순 크롤러로는 글 목록을 가져올 수 없어요. 이 파이프라인은 두 단계로 동작해요.

단계방법역할
① 글 목록 수집 requests JSON API 네이버 내부 API(/api/blogs/komx3/post-list)를 직접 호출해 전체 logNo 목록을 수집
② 본문 수집 Playwright 브라우저 각 글을 실제 Chromium으로 열어 JS 렌더링 후 본문 파싱
③ 저장 Markdown 변환 YAML frontmatter + 본문을 .md 파일로 저장
💡
--resume 옵션 덕분에 이미 수집한 글은 건너뛰어요. 새 글만 추가 추출하면 돼요.
Section 02

최초 1회 환경 설정

처음 한 번만 하면 돼요. 이후 정기 실행 때는 이 단계를 건너뛰세요.

1
프로젝트 폴더 준비

압축 파일을 풀어 작업 폴더를 만들어요.

macOS
Terminal 
mkdir -p ~/think-tank/naver-blog-scraper
unzip ~/Downloads/naver-blog-scraper-mac.zip \
      -d ~/think-tank/naver-blog-scraper
Windows
PowerShell 
mkdir ~\kom-tank\naver-blog-scraper
Expand-Archive ~\Downloads\naver-blog-scraper-win.zip `
               ~\kom-tank\naver-blog-scraper
2
가상환경 생성 및 패키지 설치

Python 패키지를 전역 환경과 분리해 설치해요.

macOS
Terminal 
cd ~/think-tank/naver-blog-scraper
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
Windows
PowerShell 
cd ~\kom-tank\naver-blog-scraper
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
⚠️
Windows에서 Activate.ps1 실행 오류가 나면:
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned 실행 후 재시도
3
Playwright 브라우저 설치 (약 200MB, 최초 1회)

본문 수집에 사용하는 Chromium 브라우저를 설치해요.

macOS / Windows 공통
Terminal / PowerShell 
playwright install chromium
✅
설치 완료 메시지 확인 후 다음 단계로. 이후엔 이 명령어 불필요.
Section 03

새 글 추출하기 (정기 실행)

환경 설정 이후엔 이것만 하면 돼요. 새 글이 쌓였을 때마다 실행하세요.

1
폴더 이동 + 가상환경 활성화
macOS
Terminal 
cd ~/think-tank/naver-blog-scraper
source venv/bin/activate
Windows
PowerShell 
cd ~\kom-tank\naver-blog-scraper
.\venv\Scripts\Activate.ps1

프롬프트 앞에 (venv) 가 붙으면 활성화 완료.

2
새 글만 추가 수집

--resume 옵션을 쓰면 이미 수집한 글은 건너뛰고 새 글만 가져와요.

macOS / Windows 공통
Terminal / PowerShell 
python3 scraper.py --resume   # macOS
python  scraper.py --resume   # Windows
💡
처음 전체 수집할 때는 --resume 없이 실행. 이후 정기 추출은 항상 --resume.
3
결과 통계 확인
macOS / Windows 공통
Terminal / PowerShell 
python3 verify.py   # macOS
python  verify.py   # Windows

연도별 글 수, 카테고리 TOP 10, 태그 TOP 20 등 통계를 확인할 수 있어요.

명령어 치트시트

상황명령어
처음 전체 수집python3 scraper.py
새 글만 추가 수집python3 scraper.py --resume
테스트 (5개만)python3 scraper.py --limit 5
중단 후 재개python3 scraper.py --resume
결과 통계 보기python3 verify.py
Section 04

결과물 확인 및 Obsidian 연동

macOS

Obsidian vault 연동

naver-archive 폴더를 Obsidian vault에 심볼릭 링크로 연결하면 글이 쌓일 때마다 자동으로 vault에 반영돼요.

macOS
Terminal 
ln -s ~/think-tank/naver-archive \
      ~/path/to/obsidian-vault/Naver-Blog
Windows
PowerShell (관리자 권한) 
New-Item -ItemType Junction `
  -Path "C:\Users\eunhy\obsidian-komx2\Naver-Blog" `
  -Target "C:\Users\eunhy\kom-tank\naver-archive"
Section 05

문제 해결

증상원인해결
글 목록 0개 네이버 API 구조 변경 python3 find_api.py 실행 후 Claude에 결과 붙여넣기
본문 수집 느림 Playwright JS 렌더링 대기 정상. 글당 약 3~5초 소요
중간에 멈춤 네트워크 오류 / 맥북 슬립 --resume 으로 재실행
(venv) 안 붙음 가상환경 미활성화 source venv/bin/activate 재실행
날짜가 unknown 날짜 파싱 실패 파일명 날짜로 수동 수정 또는 무시
이미지 안 보임 URL 링크만 저장됨 Obsidian에서 원본 URL 클릭으로 확인
⚠️
.progress.json 삭제 금지. 이 파일이 없으면 --resume 이 작동하지 않아 전체를 다시 수집해야 해요.
📌
비공개 글은 수집 불가. 공개 설정된 글만 추출돼요.
