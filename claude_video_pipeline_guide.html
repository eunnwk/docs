01

왜 영상 편집 툴보다 이 방법인가

Premiere, DaVinci Resolve는 강력하지만 블랙박스예요. 타임라인에 뭔가 올려두면 왜 그 컷을 골랐는지 어디에도 남지 않아요. 반면 이 파이프라인은 모든 결정이 텍스트 파일에 기록됩니다.

핵심 차이 — 편집 툴 타임라인은 시각적 블랙박스. 이 방식은 모든 결정이 edit_plan.json에 이유와 함께 기록되어 AI가 스스로 검증하고 재생성할 수 있어요.
♻️

재현 가능

edit_plan.json 하나만 있으면 언제든 똑같은 영상을 다시 만들 수 있어요. 1년 후에도요.

✅

AI가 검증

verified: false 필드 덕분에 '미검증 컷 있어?' 물으면 Claude가 바로 찾아줘요.

🔌

자동화 연결

JSON이니까 다른 도구와 자동으로 연결돼요. 파이프라인의 부품이 됩니다.

편집 히스토리를 diff로 추적할 수 있어요. '왜 이 컷인가'가 커밋 로그에 남아요.

03

단계별 실습 가이드

아래 7단계를 순서대로 따라하면 영상 편집 소프트웨어 없이 완성본을 만들 수 있어요.

환경 준비 — Python 3, ffmpeg, Node.js 설치 후 가상환경 만들기 
mkdir ~/fable-workflow && cd ~/fable-workflow
python3 -m venv venv && source venv/bin/activate
pip install openai-whisper
1
트랜스크립션 — 영상을 텍스트로 변환
Whisper AI가 음성을 텍스트로 바꾸고 단어별 타임스탬프를 기록해요. 각 단어가 몇 초에 시작/끝나는지 알 수 있어요. 이 데이터가 전체 파이프라인의 기반입니다.
whisper video.mp4 \
  --output_format json \
  --language ko \
  --word_timestamps True \
  --model small \
  --output_dir .
출력: video.json — 모든 편집 결정의 출발점
2
샷 선택 & 편집 계획 JSON 생성
Claude에게 트랜스크립트를 주고 '핵심 메시지 구간 TOP 5를 골라줘'라고 요청해요. Claude는 각 선택에 이유(reason)와 의도(intent)를 붙여 JSON으로 돌려줘요. 이게 편집의 설계도입니다.
// edit_plan.json 구조
{
  "project": "design_as_data_v1",
  "cuts": [
    {
      "scene": 1,
      "start": 112.0,
      "end": 120.0,
      "intent": "문제 제기",
      "reason": "기존 방식의 한계를 구체적 예시로 설명",
      "verified": false  ← AI 자동 검증 필드
    }
  ]
}
verified: false 필드 — Claude가 '미검증 컷 찾아줘' 하면 자동으로 플래그 세워요
3
ffmpeg 자동 컷팅
edit_plan.json을 읽어서 원본 영상을 자동으로 잘라요. Python 스크립트가 JSON의 start/end 값을 읽고 ffmpeg 명령어를 자동 실행합니다. 직접 명령어를 입력할 필요가 없어요.
# Claude가 생성한 컷팅 스크립트
import json, subprocess, os

with open('edit_plan.json') as f:
    plan = json.load(f)

os.makedirs('cuts', exist_ok=True)
for cut in plan['cuts']:
    i = cut['scene']
    subprocess.run([
        'ffmpeg', '-i', 'video.mp4',
        '-ss', str(cut['start']),
        '-to', str(cut['end']),
        '-c', 'copy', f'cuts/cut_{i:02d}.mp4'
    ])
출력: cuts/cut_01.mp4 ~ cut_05.mp4
4
컬러 그레이딩 — 커스텀 LUT 생성 & 적용
LUT(Look-Up Table)는 영상의 색감을 바꾸는 데이터 파일이에요. 여기선 Claude가 코드로 직접 .cube 파일을 작성해요. 프리셋이 아닌 나만의 색감을 만들 수 있어요.
# .cube LUT 적용
ffmpeg -i cut_01.mp4 \
  -vf "lut3d=luts/film_warm.cube" \
  -c:a copy \
  cut_01_graded.mp4
섀도우 청록 틴트 + 하이라이트 노란빛 + S-curve 필름 룩
5
Remotion — React로 모션 그래픽스
Remotion은 React 코드로 영상 그래픽을 만드는 도구예요. 자막과 타이틀 카드를 코드로 작성하면 프레임 단위로 타이밍을 제어할 수 있어요. 트랜스크립트 타임스탬프와 sync됩니다.
// 자막 컴포넌트 예시
import { useCurrentFrame, interpolate } from "remotion";

export const Subtitle = ({ text, intent }) => {
  const frame = useCurrentFrame();
  const opacity = interpolate(
    frame, [0, 15, 75, 90], [0, 1, 1, 0]
  );
  return (
    <div style={{ opacity, position: "absolute", bottom: 80 }}>
      {text}
    </div>
  );
};
타임스탬프만 바꾸면 전체 타이밍 자동 조정
6
Figma MCP — 실시간 디자인 협업 루프
Figma MCP를 Claude Code에 연결하면 디자이너가 Figma에서 색상/폰트를 수정할 때 Claude가 즉시 Remotion 컴포넌트 코드를 업데이트해요. 디자인 변경이 영상에 실시간 반영됩니다.
// .mcp.json
{
  "mcpServers": {
    "figma": {
      "command": "npx",
      "args": ["-y", "figma-developer-mcp"],
      "env": { "FIGMA_TOKEN": "your_token_here" }
    }
  }
}
루프: Figma 수정 → Claude 감지 → 코드 업데이트 → 재렌더
7
최종 합성 & 렌더링
컬러그레이딩된 컷들을 순서대로 이어붙이고 자막을 합성해요. concat_list.txt에 파일 순서를 적으면 ffmpeg이 하나의 완성본으로 합쳐줍니다.
# concat_list.txt 생성 후 이어붙이기
ffmpeg -f concat -safe 0 \
  -i concat_list.txt \
  -c copy \
  final_output.mp4
🎬 출력: final_output.mp4 — 완성!
04

Figma MCP 연동 상세 가이드

Step 5에서 Remotion 그래픽을 만들었다면, 다음은 Figma와 실시간으로 연결하는 거예요.

Figma Personal Access Token 발급

Figma 우측 상단 프로필 → Settings → Account → Personal Access Tokens → Generate new token

# 환경변수로 저장
export FIGMA_TOKEN="your_token_here"

Claude Code에 Figma MCP 서버 등록

프로젝트 루트에 .mcp.json 파일 생성

{
  "mcpServers": {
    "figma": {
      "command": "npx",
      "args": ["-y", "figma-developer-mcp"],
      "env": { "FIGMA_TOKEN": "your_token_here" }
    }
  }
}

Figma 파일 ID 확인 후 연결

URL에서 파일 ID 추출: https://figma.com/file/ABC123/...

# Claude Code에서 요청:
"Figma 파일 ABC123에서 색상 토큰 읽어서
 Remotion Subtitle.tsx 컴포넌트에 반영해줘"

실시간 루프 체험

Figma에서 색상 수정 → Claude Code가 변경사항 감지 → Remotion 컴포넌트 자동 업데이트 → npm run dev로 즉시 확인

Hermes 연결 포인트 — 이 패턴이 WORK vertical 오케스트레이터와 동일한 구조예요. 외부 도구 변경을 Claude가 감지해 코드를 자동 동기화하는 루프.

06

핵심 정리

한 줄 요약 — 영상 편집 툴은 한 번 쓰고 끝이지만, 이 방식은 자동화 파이프라인의 부품이 됩니다. 오늘 만든 edit_plan.json이 Hermes와 연결될 때 진짜 힘을 발휘해요.
모든 편집 결정이 JSON에 기록 → AI가 스스로 검증하고 재생성 가능
텍스트/코드/JSON 기반 → Git 버전 관리, diff, 롤백 자유
Remotion = 코드로 된 모션 그래픽 → 타임스탬프만 바꾸면 자동 sync
Figma MCP = 디자인 변경이 코드에 실시간 반영
Hermes 연결 시 = 명령어 한 줄로 전체 파이프라인 자동 실행
