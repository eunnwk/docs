Step 02

Anthropic API 크레딧 & 키 발급

이 단계에서 하는 것: Claude AI를 코드에서 사용하기 위한 API 키를 발급하고 크레딧을 충전한다. claude.ai 구독과는 완전히 별개로, API 사용량에 따라 요금이 부과된다. 회의록 100건 기준 약 $1~3 수준이다.
  • console.anthropic.com 접속 후 로그인
  • 왼쪽 메뉴 "Plans & Billing" 클릭 → 크레딧 충전
  • 왼쪽 메뉴 "API Keys" 클릭
  • "Create Key" 버튼 클릭 → 키 이름 입력 → 생성
  • 발급된 키(sk-ant- 로 시작)를 안전한 곳에 메모 — 다시 볼 수 없음
⚠️

API 키는 발급 직후에만 전체 값이 표시된다. 반드시 메모장 등에 복사해두어야 한다.

Step 03

Google Cloud Console 설정

이 단계에서 하는 것: Python 코드가 Google Docs를 읽어올 수 있도록 Google의 허가(인증)를 설정한다. Google Cloud에 앱을 등록하고, 인증 파일(credentials.json)을 발급받는다.

3-1. Google Cloud 프로젝트 생성

  • console.cloud.google.com 접속 — 회의록이 있는 구글 계정으로 로그인
  • 상단 "프로젝트 선택" 클릭 → 팝업 우측 상단 "새 프로젝트" 클릭
  • 프로젝트 이름: nx-meeting-refiner 입력 → "만들기"

3-2. API 활성화

Google Drive API와 Google Docs API 두 가지를 모두 활성화해야 한다. Drive API는 파일 목록 조회, Docs API는 문서 내용 읽기에 사용된다.
  • 왼쪽 메뉴 "API 및 서비스" → "라이브러리" 클릭
  • 검색창에 Google Drive API 입력 → 클릭 → "사용"
  • 다시 라이브러리로 돌아가서 Google Docs API 검색 → 클릭 → "사용"

3-3. OAuth 동의 화면 설정

코드가 내 Google 계정에 접근할 때 표시할 앱 정보를 설정하는 단계다. 본인만 사용하는 용도이므로 간단하게 설정하면 된다.
  • 주소창에 https://console.cloud.google.com/auth/overview 입력해서 이동
  • "시작하기" 버튼 클릭
  • 앱 이름: nx-meeting-refiner 입력
  • 사용자 지원 이메일: 드롭다운에서 본인 이메일 선택
  • "다음"을 계속 클릭해서 완료 → "만들기"
💡

하단에 "OAuth 구성 생성됨" 토스트 메시지가 뜨면 성공이다.

3-4. OAuth 클라이언트 ID 발급

코드가 Google 계정에 로그인할 때 사용하는 인증 자격증명을 만드는 단계다. 이 단계에서 만들어지는 credentials.json 파일이 핵심이다.
  • 화면에서 "OAuth 클라이언트 만들기" 버튼 클릭
  • 애플리케이션 유형: "데스크톱 앱" 선택
  • 이름: nx-meeting-refiner-client 입력 → "만들기"
  • 생성 완료 후 왼쪽 메뉴 "클라이언트" 클릭 → 목록에서 ⬇ 다운로드 아이콘 클릭
  • 다운로드된 파일 이름을 credentials.json 으로 변경
⚠️

다운로드 버튼이 작동하지 않는 경우: 클라이언트 상세 페이지에서 클라이언트 ID와 클라이언트 보안 비밀번호를 복사해두고, 아래 내용을 텍스트 에디터에 붙여넣어 직접 파일을 만든다.

credentials.json 
{
  "installed": {
    "client_id": "여기에_클라이언트ID",
    "client_secret": "여기에_보안비밀번호",
    "auth_uri": "https://accounts.google.com/o/oauth2/auth",
    "token_uri": "https://oauth2.googleapis.com/token",
    "auth_provider_x509_cert_url": "https://www.googleapis.com/oauth2/v1/certs",
    "redirect_uris": ["http://localhost"]
  }
}

3-5. 테스트 사용자 추가

외부 앱으로 등록했기 때문에 본인 계정을 테스트 사용자로 등록해야 Google 로그인이 가능하다.
  • 왼쪽 메뉴 "대상" 클릭
  • "ADD USERS" 클릭 → 본인 구글 계정 이메일 입력 → "저장"

3-6. 파일 정리

  • 바탕화면에 report-bot 폴더 생성
  • credentials.json 을 report-bot 폴더 안으로 이동
  • meeting_refiner.py 도 같은 폴더 안으로 이동
폴더 구조
바탕화면/
└── report-bot/
    ├── credentials.json   ← Google 인증 파일
    └── meeting_refiner.py ← 정제 스크립트
Step 04

Python 환경 설정

이 단계에서 하는 것: 스크립트 실행에 필요한 Python 패키지를 설치한다. 가상환경(venv)은 이 프로젝트에서만 사용하는 독립된 Python 공간으로, 다른 프로젝트와 충돌을 방지한다.

터미널(Terminal.app)을 열고 아래 명령어를 순서대로 실행한다.

bash 
# 1. report-bot 폴더로 이동
cd /Users/사용자이름/Desktop/report-bot

# 2. 가상환경 생성 (최초 1회만)
python3 -m venv venv

# 3. 가상환경 활성화
source venv/bin/activate
# → 프롬프트 앞에 (venv) 가 붙으면 성공

# 4. 필요한 패키지 설치
pip install google-auth google-auth-oauthlib google-api-python-client anthropic
✅

설치가 완료되고 프롬프트 앞에 (venv) 가 표시되면 준비 완료다.

⚠️

터미널을 새로 열 때마다 source venv/bin/activate 를 실행해야 한다. 이 명령어 없이 스크립트를 실행하면 패키지를 찾지 못해 오류가 난다.

Step 05

스크립트 설정 및 실행

이 단계에서 하는 것: meeting_refiner.py 상단의 설정값을 본인 환경에 맞게 확인하고, 테스트 실행 → 전체 실행 순서로 진행한다.

설정값 확인

meeting_refiner.py 상단에서 아래 항목이 본인 환경과 맞는지 확인한다.

meeting_refiner.py — 설정 영역 
TEST_MODE        = True   # 처음엔 반드시 True로 시작
CREDENTIALS_PATH = "/Users/사용자이름/Desktop/report-bot/credentials.json"
TOKEN_PATH       = "/Users/사용자이름/Desktop/report-bot/token.json"
DOC_ID           = "구글_문서_ID"   # 아래 방법으로 확인
VAULT_PATH       = Path("/Users/사용자이름/Documents/Obsidian Vault")
MEETING_BASE     = VAULT_PATH / "CLT/NX/회의록"
ISSUE_BASE       = VAULT_PATH / "CLT/NX/분석/이슈트래킹"
💡

구글 문서 ID 확인 방법: 구글 문서를 열고 주소창을 확인한다. /d/ 와 /edit 사이의 긴 문자열이 문서 ID다.
예: docs.google.com/document/d/1YJ08tl0HQpi.../edit

테스트 실행 (처음에는 반드시 먼저)

전체를 한 번에 실행하면 API 비용이 많이 발생할 수 있다. TEST_MODE = True 상태에서 탭 1개만 처리해 결과를 먼저 확인한다.
bash 
cd /Users/사용자이름/Desktop/report-bot
source venv/bin/activate
export ANTHROPIC_API_KEY="sk-ant-여기에_API키_입력"
python meeting_refiner.py
  • 처음 실행 시 브라우저에 Google 로그인 창이 열린다 → 본인 계정으로 로그인
  • 터미널에 🧪 테스트 모드 완료 메시지가 뜨면 성공
  • Obsidian에서 생성된 파일을 열어 포맷과 내용이 올바른지 확인
✅

결과물이 만족스러우면 전체 실행으로 넘어간다. 이상하면 스크립트 상단의 프롬프트를 수정한다.

전체 실행

테스트 확인 후 TEST_MODE 를 False 로 바꾸고 전체를 실행한다.

bash 
# TEST_MODE를 False로 변경
sed -i '' 's/TEST_MODE        = True/TEST_MODE        = False/' meeting_refiner.py

# 전체 실행
python meeting_refiner.py
💡

탭이 많으면 시간이 걸린다. 터미널에 각 탭의 처리 진행 상황이 출력되며, 마지막에 🎉 완료! 가 뜨면 전체 처리가 끝난 것이다.

Step 06

탭 분류 설정

이 단계에서 하는 것: 스크립트가 Google Docs의 각 탭을 타입 1(회의록) 또는 타입 2(이슈트래킹)로 자동 분류할 수 있도록 설정한다. 탭 구조가 바뀌거나 새 탭이 생겼을 때도 여기를 수정한다.

탭 구조 확인 방법

스크립트 실행 전에 실제 탭 계층 구조를 확인하고 싶다면 아래 명령어를 실행한다.

bash 
python -c "
from googleapiclient.discovery import build
from google.oauth2.credentials import Credentials
creds = Credentials.from_authorized_user_file('token.json')
service = build('docs', 'v1', credentials=creds)
doc = service.documents().get(documentId='여기에_문서ID', includeTabsContent=True).execute()
def print_tabs(tabs, depth=0):
    for tab in tabs:
        props = tab.get('tabProperties', {})
        print('  ' * depth + '[' + props.get('title', '') + ']')
        print_tabs(tab.get('childTabs', []), depth+1)
print_tabs(doc.get('tabs', []))
"

분류 설정 방법

meeting_refiner.py 상단에서 아래 두 변수를 수정한다.

meeting_refiner.py — 분류 설정 
# 타입 1로 처리할 부모 탭 이름 (연도 그룹)
YEAR_PARENTS = {"2024", "2025", "2026 회의록"}

# 타입 2로 강제 지정할 탭 이름 (이름 기준)
ISSUE_TABS = {
    "대응 2026",
    "대응",
    "ChatBot 업무 타임라인",
    "DS w/CLV",
    "AI Chat Fomatter",
    "2024 History",
    "NX 개발 배경 및 현황",
    "유저 리서치 & 페르소나",
}
상황할 일
새 연도 그룹 탭이 생김 (예: 2027 회의록) YEAR_PARENTS 에 추가
이슈트래킹 성격의 탭이 새로 생김 ISSUE_TABS 에 탭 이름 추가
탭이 건너뛰어짐 (분류 불가) 탭의 부모 이름을 확인하고 위 두 변수 중 적절한 곳에 추가

Obsidian에서 인사이트 조회

Dataview 플러그인을 사용하면 저장된 회의록을 다양한 방법으로 조회할 수 있다.

dataview — 액션아이템 미완료 조회 
TASK
FROM "CLT/NX/회의록"
WHERE !completed
dataview — 프로젝트별 회의 목록 
TABLE date, meeting_type, attendees
FROM "CLT/NX/회의록"
WHERE project = "프로젝트명"
SORT date DESC
dataview — 이슈트래킹 목록 
TABLE topic, period_start, period_end
FROM "CLT/NX/분석/이슈트래킹"
SORT period_end DESC
