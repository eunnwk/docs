Think Tank 쓰기

GitHub로 연동된 think-tank를 어느 컴퓨터에서든 이어서 사용하는 방법

구조 이해

두 컴퓨터 사이에 GitHub가 중간 다리 역할을 해요.

sync 구조
맥북프로 (회사)
    ↕  git push / pull
  GitHub (eunnwk/think-tank)
    ↕  git push / pull
윈도우 데스크탑 (집)
💡
작업 시작 전 git pull, 끝난 후 git push — 이 두 가지만 기억하면 돼요.

시나리오 A — 맥북프로 (이미 세팅됨)

💻
매일 사용 루틴
회사 맥북프로에서 쓸 때

작업 시작할 때

터미널 
# 1. think-tank 폴더로 이동
cd ~/think-tank

# 2. 다른 컴퓨터 내용 받기
git pull

# 3. Claude Code 실행
claude

작업 끝날 때

터미널 
# 오늘 내용 저장 후 올리기
git add .
git commit -m "오늘 작업"
git push

시나리오 B — 새 컴퓨터에서 처음 세팅

집 윈도우 데스크탑처럼 처음 쓰는 컴퓨터라면 아래 순서대로 한 번만 하면 돼요.

🖥️
처음 한 번만 — 클론
새 컴퓨터 최초 세팅
  1. Claude Code 설치
    터미널 
    npm install -g @anthropic-ai/claude-code
    💡
    npm이 없다고 나오면 nodejs.org에서 Node.js 먼저 설치
  2. GitHub에서 think-tank 가져오기
    터미널 
    git clone https://github.com/eunnwk/think-tank.git
cd think-tank
  3. Claude Code 실행
    터미널 
    claude
    💡
    "trust this folder?" 물으면 1 입력 후 Enter
✅
클론 이후부터는 맥북과 똑같이 git pull 시작 / git push 마무리 루틴으로 쓰면 돼요.

클론 이후 매일 루틴

터미널 — 작업 시작 
# 폴더 이동 (클론한 위치 기준)
cd think-tank
git pull
claude
터미널 — 작업 끝 
git add .
git commit -m "오늘 작업"
git push

자주 겪는 상황

⚠️
git push 할 때 로그인 요청이 뜨면?
GitHub 인증 오류

GitHub에서 Personal Access Token을 발급받아야 해요.

  1. github.com → Settings → Developer settings → Personal access tokens → Tokens (classic)
  2. Generate new token → repo 체크 → 생성
  3. 비밀번호 입력 창에 토큰 붙여넣기
🔄
git pull 할 때 충돌이 나면?
두 컴퓨터에서 같은 파일을 동시에 수정했을 때

Claude Code에 그대로 말하면 해결해줘요.

Claude Code에 입력
git pull 하다가 충돌 났어. 해결해줘.
📁
think-tank 폴더 위치가 헷갈리면?
폴더 찾기
터미널 
# 맥
find ~ -name "think-tank" -type d 2>/dev/null
윈도우 PowerShell 
Get-ChildItem -Path $HOME -Recurse -Directory -Filter think-tank
