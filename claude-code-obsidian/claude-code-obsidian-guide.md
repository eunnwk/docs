# Claude Code × Obsidian 연결 가이드
> 터미널을 한 번도 써본 적 없어도 따라할 수 있는 단계별 가이드

---

## 이 작업이 왜 유용한가?

### 기존 AI 챗봇의 한계
일반 Claude.ai나 ChatGPT는 **대화가 끝나면 모든 내용을 잊습니다.**  
매번 배경 설명을 다시 해야 하고, 내가 오랫동안 쌓아온 메모나 아이디어는 활용되지 못합니다.

### Claude Code + Obsidian으로 달라지는 것

| 기존 방식 | Claude Code + Obsidian |
|---|---|
| 대화가 끝나면 기억 초기화 | 내 Vault의 모든 파일을 직접 읽음 |
| 매번 맥락 설명 필요 | 수년간 쌓인 메모를 바탕으로 답변 |
| 단순 정보 검색 수준 | 내 생각의 흐름과 패턴까지 파악 |
| 외부 지식 기반 | 내 머릿속 지식이 AI의 연료 |

### 실제로 가능한 것들
- `"최근 1주일 메모를 읽고 오늘 할 일을 정리해줘"`
- `"내 문체와 논리 구조를 파악해서 블로그 초안 써줘"`
- `"내 메모 중 서로 연관된 아이디어를 찾아서 연결해줘"`
- `"UX 관련 메모들만 모아서 요약 노트 만들어줘"`
- `"특정 주제에 대해 내가 6개월간 어떻게 생각이 변했는지 분석해줘"`

**메모가 쌓일수록 AI 분석의 정확도가 기하급수적으로 올라갑니다.**  
Obsidian이 데이터 저장소라면, Claude Code는 그 데이터를 구동하는 엔진입니다.

---

## 준비물

- Mac (이 가이드는 맥북 기준)
- Anthropic 계정 (claude.ai 계정)
- Obsidian 설치 및 Vault 폴더 위치 확인
- Claude Pro 플랜 (Claude Code 사용 필요)

---

## Step 1. Node.js 설치 확인

Claude Code를 설치하려면 Node.js(버전 18 이상)가 필요합니다.

### 터미널 열기
1. 키보드에서 `⌘ (Command) + Space` 동시에 누르기
2. 검색창에 `terminal` 입력
3. Enter 키 누르기
4. 검은색(또는 흰색) 창이 열리면 성공

### Node.js 설치 여부 확인
터미널에 아래를 입력하고 Enter:

```
node -v
```

- `v20.0.0` 같은 숫자가 나오면 → 이미 설치됨, Step 2로 이동
- `command not found`가 나오면 → 아래 설치 진행

### Node.js 설치 방법
1. 브라우저에서 https://nodejs.org 접속
2. **LTS** 버튼 클릭해서 다운로드
3. 다운받은 파일 실행 → 계속 → 설치 완료
4. 터미널 **새로 열고** 다시 `node -v` 입력해서 확인

---

## Step 2. Claude Code 설치

터미널에 아래를 입력하고 Enter:

```
npm install -g @anthropic-ai/claude-code
```

설치에 1~2분 정도 걸립니다. 완료 메시지가 나오면 다음으로.

> ⚠️ **오류가 나는 경우**: 앞에 `sudo`를 붙여서 입력하세요.  
> `sudo npm install -g @anthropic-ai/claude-code`  
> 비밀번호 입력 요청이 오면 맥 로그인 비밀번호를 입력합니다. (입력해도 화면에 표시되지 않는 게 정상)

---

## Step 3. Anthropic 계정 로그인

```
claude login
```

입력하면 브라우저 창이 자동으로 열립니다.  
Claude.ai에 로그인하면 터미널로 돌아와 인증이 완료됩니다.

---

## Step 4. Obsidian Vault 폴더로 이동

터미널에서 Obsidian Vault 폴더 안으로 이동해야 합니다.

### 방법 1. 드래그 앤 드롭 (가장 쉬움)
1. 터미널에 `cd ` 입력 (cd 뒤에 스페이스 하나)
2. Finder에서 Obsidian Vault 폴더를 터미널 창 위로 **드래그**
3. 경로가 자동 입력됨
4. Enter 키 누르기

### 방법 2. 직접 입력
Vault가 Documents 폴더 안에 있다면:

```
cd ~/Documents/Obsidian\ Vault
```

> 💡 폴더 이름에 공백이 있으면 `\`(백슬래시)를 공백 앞에 넣어야 합니다.

---

## Step 5. Claude Code 실행

```
claude
```

입력하면 아래와 같은 화면이 나타납니다:

```
Welcome back eunn.wk!
─────────────────────
Sonnet 4.6 · Claude Pro
~/Documents/Obsidian Vault

> _
```

**`>`** 프롬프트가 보이면 준비 완료입니다.

> 💡 처음 실행 시 파일 접근 권한을 묻는 메시지가 나올 수 있습니다.  
> `y`를 누르고 Enter 하면 됩니다.

---

## Step 6. 첫 번째 테스트

### 파일 생성 테스트
`>` 뒤에 입력:

```
오늘 날짜로 "테스트메모" 라는 마크다운 파일을 만들고, 내용에는 내가 자주 하는 일 3가지를 예시로 넣어줘
```

Obsidian 앱에서 새 파일이 생성되었는지 확인하세요.

### 파일 읽기 테스트

```
내 Vault에 있는 파일 목록을 보여줘
```

```
테스트메모 파일 내용을 요약해줘
```

파일 목록이 보이고 내용을 읽어온다면 **연결 성공**입니다.

## Step 7. Notion 메모 이전 (선택사항)

노션에 메모가 쌓여있다면 Obsidian으로 가져올 수 있습니다.

### 노션에서 내보내기
1. 노션 왼쪽 하단 **Settings** 클릭
2. **Settings → Export all workspace content**
3. 형식: **Markdown & CSV** 선택
4. Export 클릭 → zip 파일 다운로드

### Obsidian으로 가져오기
1. 다운받은 zip 파일 압축 풀기
2. 압축 풀린 폴더 안의 `.md` 파일들을 Obsidian Vault 폴더에 붙여넣기

### 파일명 자동 정리
노션 export 파일명에는 뒤에 고유 ID 코드가 붙습니다.  
예: `회의록 a1b2c3d4e5f6.md`

Claude Code에서 자동으로 정리:

```
모든 마크다운 파일명 뒤에 붙어있는 32자리 영문숫자 코드를 제거해줘. 파일명만 바꾸고 내용은 건드리지 마.
```

## 주의사항

- Claude Code는 **텍스트 기반 `.md` 파일**은 잘 읽지만, 메모 속 이미지나 PDF 내부 텍스트는 인식하지 못할 수 있습니다.
- 폴더 구조가 복잡하거나 파일명이 모호하면 엉뚱한 파일을 참조할 수 있습니다. **파일명에 핵심 키워드**를 넣어두세요.
- Claude Code 종료는 `Ctrl + C` 또는 `/exit` 입력.
- 다음에 다시 사용할 때는 터미널에서 Vault 폴더로 이동 후 `claude` 입력하면 됩니다.

---

## 매번 실행하는 방법 (요약)

```bash
# 1. 터미널 열기 (⌘ + Space → terminal)
# 2. Vault 폴더로 이동
cd ~/Documents/Obsidian\ Vault
# 3. Claude Code 실행
claude
```
