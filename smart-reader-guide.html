이 가이드가 만드는 것

복잡한 '편집(수정)' 기능은 과감히 제외하고, 사용자가 글을 읽는 데 집중할 수 있도록 다음 기능만 넣은 읽기 전용 뷰어입니다.

  • txt / md 파일 저장
  • 밑줄(하이라이트) — 텍스트를 드래그하면 표시
  • 이어읽기 — 스크롤 위치 기억
  • 공유
Milestone 01
프로젝트 기획 및 준비 (PRD 작성과 환경 구축)
1
기능의 범위 정하기 (기획)

무작정 코딩을 시작하면 배가 산으로 갑니다. 개발할 기능의 명확한 선을 긋는 작업입니다. 이번 프로젝트에서는 복잡한 '편집(수정)' 기능은 과감히 제외하고, '저장, 밑줄(하이라이트), 이어읽기, 공유' 기능만 넣기로 약속했습니다.

2
리액트(React) 방 만들기

리액트는 요즘 웹사이트를 만들 때 가장 많이 쓰는 '조립식 건물 자재'입니다. 아래 명령어로 리더기 웹앱을 위한 빈 방(폴더)과 기초 뼈대를 자동으로 짓습니다.

npx create-react-app smart-reader
3
필요한 부품(라이브러리) 설치하기

빈 방에 가구와 가전제품을 들여놓는 과정입니다. npm install 명령어로 두 가지 핵심 부품을 설치합니다.

  • lucide-react — 돋보기·책갈피·공유 모양의 고품질 아이콘 부품
  • react-markdown — 마크다운(.md)의 특수기호(#, **, -)를 예쁜 글씨/표로 바꿔주는 번역기 부품
npm install lucide-react react-markdown
Milestone 02
Cursor 에디터를 통한 프로토타입 제작
4
Cursor 에디터 켜고 방 열기

메모장 같은 코드 편집 프로그램인 'Cursor'를 실행한 뒤, 아까 만든 smart-reader 폴더를 엽니다. 터미널에 code .을 치면 현재 위치에서 바로 Cursor가 열립니다.

cd smart-reader code .
5
AI 작가(Composer)에게 코딩 주문하기

Cursor의 핵심 기능인 AI 작가(Composer)를 깨우는 작업입니다. Cmd + I (윈도우는 Ctrl + I)를 누르면 AI 대화창이 뜹니다. 여기에 아래처럼 주문서를 넣습니다.

텍스트를 드래그하면 하이라이트가 쳐지고, 스크롤 위치를 기억하는 읽기 전용 뷰어를 만들어줘.
6
AI가 짠 코드 수락하고 확인하기

AI가 작성해 준 코드를 검토하고 Accept(수락) 버튼을 눌러 실제 src/App.js 파일에 덮어씌웁니다. 그 후 아래 명령으로 내 컴퓨터 안에 임시 웹서버를 켜고 브라우저 화면이 제대로 뜨는지 눈으로 확인합니다.

npm start
Milestone 03
깃허브(GitHub)에 안전하게 대피 및 퇴근하기
7
깃허브에 온라인 보관함 만들기

컴퓨터가 고장 나거나 다른 장소에서 일할 때를 대비해, 인터넷 세상(GitHub.com)에 프로젝트 전용 안전 금고(Repository)를 하나 개설하고 금고 주소를 복사해 둡니다.

8
내 컴퓨터와 금고 연결하고 첫 업로드하기

내 컴퓨터의 코드를 포장해서 방금 만든 인터넷 금고로 보내는 과정입니다. 터미널에 아래 명령어들을 순서대로 입력합니다.

git init # 1. 이 폴더에서 버전 관리(Git) 시작 선언 git add . # 2. 모든 파일을 업로드 장바구니에 담기 git commit -m "feat: 첫 프로토타입" # 3. 꼬리표를 달아 상자에 포장 git branch -M main # 4. 메인 브랜치 이름을 main으로 통일 git remote add origin [금고주소] # 5. 내 컴퓨터와 깃허브 금고 연결 git push -u origin main # 6. 금고 안으로 최종 업로드!
Milestone 04
다음 작업 이어하기 (출근 루틴)
9
다음 날 같은 컴퓨터에서 이어서 할 때

어제 하던 컴퓨터를 다시 켠 경우, 코드가 그대로 남아있으므로 복잡한 연결 과정이 필요 없습니다. Cursor를 켜고 아래 명령으로 화면을 띄운 뒤 바로 코딩을 시작합니다.

npm start
작업이 끝나 오늘 분량을 깃허브에 누적 저장할 때 — 딱 3줄
git add . git commit -m "style: 디자인 수정" git push
10
아예 다른 새 컴퓨터에서 이어서 할 때

노트북을 새로 샀거나 카페 컴퓨터에서 작업할 때입니다. 깃허브 금고에 코드가 보관되어 있으므로 몸만 이동하면 됩니다.

git clone [내 깃허브 금고 주소] # 1. 코드 통째로 다운로드 cd smart-reader # 2. 다운로드된 폴더로 들어가기 npm install # 3. 깃허브에 안 올렸던 부품들 자동 재설치 npm start # 4. 웹 화면 켜고 이어서 작업
💡 시니어 개발자의 한마디
비전공자분들이 개발을 포기하는 가장 큰 이유는 코딩 문법이 아니라, 이러한 '파일 관리'와 '깃허브 업로드' 단계에서 에러가 났을 때 해결 방법을 모르기 때문입니다. 이 문서의 순서대로만 움직이면 소중한 코드를 잃어버리는 일 없이 안전하게 개발을 이어 나갈 수 있습니다!

핵심

기능은 읽기에 필요한 것만 — 저장·하이라이트·이어읽기·공유.
코드는 매번 깃허브에 누적 저장해서 어디서든 이어 작업.

기획 → Cursor로 제작 → 깃허브 백업 → 다음 날 이어하기. 이 루틴만 지키면 비전공자도 끝까지 완주할 수 있다.
