Claude Code × Figma Plugin — 실전 제작 가이드
Figma Plugin 완성됨 2026.06.15

Frame Namer
제작 가이드

Claude Code로 Figma 플러그인을 처음부터 만든 전 과정. 규칙 문서 작성부터 실제 Figma 로드까지.

만든 것
레이어 자동 이름변경 플러그인
사용 도구
Claude Code + Figma Plugin API
핵심 파일
manifest.json, code.js, ui.html
✦ 시작 전 알아두기 — .claude가 아니라 그냥 claude

.claude는 규칙·스킬 문서를 담아두는 폴더 이름이고, 터미널에서 실제로 입력하는 명령어는 점 없는 claude다. 프로젝트 폴더 안에서 claude를 실행하면 Claude Code가 그 폴더 안의 파일들(CLAUDE.md, PRD.md, plugin/ 등)을 바로 확인할 수 있다. 즉, 폴더에 cd로 들어간 다음 claude만 입력하면 된다.

1

Phase 1 — 폴더 구조 & 규칙 문서 작성

코드를 짜기 전에 Claude Code에게 줄 "지침서"를 먼저 만든다

⏱ 15분
✦ 왜 문서부터 먼저 만드나?

Claude Code는 프로젝트 폴더의 CLAUDE.md를 가장 먼저 읽는다. 규칙을 여기 써두면 "TypeScript 쓰지 마", "PRD에 없는 기능 추가하지 마" 같은 제약을 코드 생성 내내 지킨다. 문서 없이 시작하면 Claude가 멋대로 복잡하게 만들 수 있다.

1

폴더 구조 생성

터미널에서 프로젝트 폴더를 만들고 그 안에서 하위 폴더를 생성한다. 순서가 중요하다 — cd 로 폴더 안에 들어간 다음 파일을 만들어야 한다.

Terminal 
cd ~/Desktop
mkdir figma-frame-namer
cd figma-frame-namer          # ← 반드시 폴더 안으로 이동 후 실행
mkdir -p .claude/skills
touch .claude/CLAUDE.md
touch .claude/skills/SKILL-figma-plugin.md
touch PRD.md
⚠ 실수 포인트

처음에 cd figma-frame-namer 없이 Desktop에서 바로 mkdir -p .claude/skills를 실행했더니 .claude/ 폴더가 Desktop에 생겼다. 항상 폴더 안에 들어간 다음 실행할 것.

완성된 구조 (Finder에서는 .claude/가 숨김 처리되어 안 보임 — 정상):

figma-frame-namer/ ├── .claude/ ← Cmd+Shift+. 로 Finder에서 표시 │ ├── CLAUDE.md ← Claude Code가 시작 시 자동으로 읽음 │ └── skills/ │ └── SKILL-figma-plugin.md └── PRD.md
2

PRD.md 작성 — 무엇을 만들지 정의

PRD(Product Requirements Document)는 "무엇을 만드는지" 정의하는 문서다. Claude Code가 이걸 읽고 기능 범위를 판단한다. 없으면 임의로 기능을 추가한다.

Terminal — PRD.md 내용 작성 
cat > PRD.md << 'EOF'
# Frame Namer — Product Requirements Document

## 제품 개요
Figma에서 선택한 레이어를 정해진 네이밍 규칙에 따라
자동으로 이름 변경해주는 플러그인

## 핵심 기능 (MVP)
1. 선택한 레이어 목록 표시
2. 네이밍 패턴 입력 (예: "Card/{index}" → Card/1, Card/2...)
3. 미리보기 후 적용
4. 되돌리기 지원 (Figma 기본 Undo)

## 비기능 요구사항
- UI는 단순하게: 입력창 1개 + 버튼 2개
- 에러 처리: 선택 없을 때 안내 메시지
- 코드는 TypeScript 없이 순수 JavaScript로

## 사용 흐름
1. 레이어 선택 (1개 이상)
2. 플러그인 실행
3. 패턴 입력 (기본값: "Frame/{index}")
4. [미리보기] → 결과 확인
5. [적용] → 실제 이름 변경
EOF

CLAUDE.md 작성 — 개발 규칙 정의

Claude Code가 이 프로젝트를 열 때 가장 먼저 읽는 파일. 여기서 "금지사항"과 "규칙"을 정해두면 코드 생성 내내 지킨다. 나중에 코드 리뷰할 때 이 기준으로 검토시킬 수 있다.

Terminal — CLAUDE.md 내용 작성 
cat > .claude/CLAUDE.md << 'EOF'
# CLAUDE.md — Frame Namer 프로젝트 지침

## 역할
너는 Figma 플러그인 개발 전문가다.
PRD.md와 스킬 문서를 반드시 먼저 읽고 개발을 시작한다.

## 필수 참조 문서
- PRD.md: 제품 요구사항
- .claude/skills/SKILL-figma-plugin.md: Figma API 패턴

## 개발 규칙
1. 코드 작성 전 반드시 계획을 먼저 말하고 승인을 받는다
2. TypeScript 사용 금지 — 순수 JavaScript만 사용
3. 외부 라이브러리 사용 금지 — Figma Plugin API만 사용
4. 파일은 plugin/ 폴더 안에 생성한다

## 금지사항
- PRD에 없는 기능 임의 추가 금지
- UI를 복잡하게 만들지 않기
- 빌드 과정이 필요한 도구(webpack, npm 등) 사용 금지
EOF
4

SKILL-figma-plugin.md 작성 — Figma API 지식 주입

스킬 문서는 Claude Code에게 "이 도메인의 패턴과 규칙"을 알려준다. Figma Plugin API의 핵심 패턴을 여기 써두면 Claude가 처음부터 올바른 코드를 생성한다. 한 번 작성하면 다른 플러그인 프로젝트에도 재사용 가능하다.

Terminal — SKILL 문서 작성 
cat > .claude/skills/SKILL-figma-plugin.md << 'EOF'
# SKILL: Figma Plugin 개발

## 필수 파일 구조
plugin/
├── manifest.json
├── code.js
└── ui.html

## manifest.json 규칙
{
  "name": "플러그인 이름",
  "id": "숫자 12자리",
  "api": "1.0.0",
  "main": "code.js",
  "ui": "ui.html",
  "editorType": ["figma"]   ← 반드시 포함
}

## code.js 핵심 패턴
figma.showUI(__html__, { width: 320, height: 380 });
figma.ui.postMessage({ type: 'SELECTION', data: [...] });
figma.ui.onmessage = (msg) => { ... };
figma.closePlugin();

## ui.html 핵심 패턴
// Figma로 전송 — pluginMessage 래핑 필수!
parent.postMessage({ pluginMessage: { type: 'APPLY' } }, '*');

// Figma로부터 수신
window.onmessage = (e) => {
  const msg = e.data.pluginMessage;
};

## 자주 하는 실수
1. editorType 누락 → Manifest error
2. pluginMessage 래핑 누락 → 메시지 전달 안됨
3. manifest에 icon 필드 추가 → Unexpected property 에러
EOF
✦ SKILL 문서의 핵심 가치

오늘 실제로 겪은 에러들(editorType 누락, icon 필드 에러)을 SKILL 문서에 "자주 하는 실수"로 추가해뒀다. 다음에 다른 플러그인 만들 때 이 문서 덕분에 같은 실수를 반복하지 않는다.

2번 프롬프트 — manifest.json + code.js 생성

Claude Code 프롬프트 #2 
좋아, 계획대로 진행해줘.
Step 1: plugin/manifest.json 먼저 만들어줘.
Step 2: plugin/code.js 만들어줘.

Claude가 생성한 code.js 핵심 로직:

plugin/code.js — 핵심 구조 
// UI 열기
figma.showUI(__html__, { width: 320, height: 380 });

var selection = figma.currentPage.selection;

// 선택 없을 때 안내 후 종료
if (selection.length === 0) {
  figma.notify('레이어를 1개 이상 선택해주세요.');
  figma.closePlugin();
} else {
  // 선택된 레이어 이름 목록을 UI로 전달
  figma.ui.postMessage({
    type: 'SELECTION',
    count: selection.length,
    names: selection.map(function(n) { return n.name; })
  });
}

// {index} 패턴을 번호로 변환
function applyPattern(pattern, index) {
  return pattern.replace('{index}', index + 1);
}

// UI 메시지 처리
figma.ui.onmessage = function(msg) {
  if (msg.type === 'PREVIEW') {
    var previews = selection.map(function(node, i) {
      return applyPattern(msg.pattern, i);
    });
    figma.ui.postMessage({ type: 'PREVIEW_RESULT', previews: previews });
  }
  if (msg.type === 'APPLY') {
    selection.forEach(function(node, i) {
      node.name = applyPattern(msg.pattern, i);
    });
    figma.closePlugin();
  }
  if (msg.type === 'CANCEL') figma.closePlugin();
};

3번 프롬프트 — ui.html 생성

Claude Code 프롬프트 #3 
이제 plugin/ui.html을 만들어줘.
- 선택된 레이어 개수 표시
- 패턴 입력 필드 (기본값: "Frame/{index}")
- [미리보기] 버튼 → 변경될 이름 목록 표시
- [적용] 버튼 → code.js로 메시지 전송
- 스타일은 Figma UI와 어울리게 심플하게
✦ ui.html의 핵심 통신 코드

4번 프롬프트 — CLAUDE.md 기준 코드 리뷰

코드가 다 만들어진 후 CLAUDE.md 규칙을 기준으로 스스로 검토하게 한다. 이 단계에서 실제로 PRD에 없는 기능이 발견되어 제거됐다.

Claude Code 프롬프트 #4 
CLAUDE.md의 규칙을 기준으로 지금까지 만든 코드를 검토해줘.
1. TypeScript 사용 여부
2. 외부 라이브러리 사용 여부
3. 에러 케이스 처리 여부
4. PRD에 없는 기능 추가 여부
문제 있으면 바로 수정해줘.

리뷰 결과 발견된 문제와 수정 내용:

❌ 발견된 문제
PRD에 없는 패턴 변수가 코드에 추가됨: - {i} (0-based index) - {name} (원래 이름) - UI에 hint 텍스트 표시
✅ 수정 결과
PRD에 명시된 {index}만 남김. applyPattern() 함수에서 {i}, {name} 처리 코드 제거. hint 텍스트와 CSS도 제거.
✦ 이 단계의 의미

CLAUDE.md에 "PRD에 없는 기능 임의 추가 금지"를 써뒀기 때문에 Claude 스스로 위반을 찾아내고 수정했다. 사람이 코드 한 줄씩 읽지 않아도 규칙 기반으로 검수가 된다.

Phase 3 — Figma 로드 & 테스트

만든 플러그인을 Figma 데스크탑 앱에서 실제로 실행

⏱ 15분
1

Figma에 플러그인 등록

⚠ 반드시 데스크탑 앱 사용

figma.com 웹 버전은 로컬 플러그인 로드가 불가능하다. Figma 데스크탑 앱에서만 가능하다.

Figma 데스크탑 앱에서:

  1. 아무 Figma 파일 열기
  2. 상단 메뉴 → Plugins → Development → Import plugin from manifest...
  3. Desktop/figma-frame-namer/plugin/manifest.json 선택
  4. 플러그인 목록에 "Frame Namer Development" 표시 확인
2

발생한 에러와 해결 방법

실제로 발생한 에러 2개와 해결 방법. 다음에 다시 만들 때 참고.

에러 1 — Missing editorType in manifest

원인: manifest.json에 editorType 필드가 없었음
해결: "editorType": ["figma"] 추가
교훈: SKILL 문서에 필수 필드로 등록해두면 다음엔 처음부터 포함됨

에러 2 — Unexpected extra property: icon

원인: manifest.json에 "icon": "icon.svg" 추가했더니 에러
해결: icon 필드 제거
교훈: 로컬 개발용 manifest에서는 icon 지원 안 됨. 커뮤니티 배포 시에만 사용
