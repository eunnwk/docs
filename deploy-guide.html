Claude → GitHub Pages
자동 배포 시스템 구축 가이드

📅 2026-03-30 ⏱ 약 20분 🏷 GitHub · Deploy · Claude Code

Claude가 만든 HTML 파일을 GitHub Pages에 반자동으로 배포하는 시스템입니다. 배포 전 미리보기를 확인하고, 승인하면 push까지 자동으로 처리됩니다. 이 가이드는 실제 세팅 과정을 그대로 담았습니다.

Overview

전체 플로우

✍️
Claude가 HTML 생성 Claude Code 또는 claude.ai에서 요청
▼
🔍
deploy.sh → 브라우저 미리보기 로컬에서 직접 확인
▼
✅
y 입력 → 승인 이 단계에서 배포 여부 결정
▼
🚀
git push → GitHub Pages 반영 약 1~2분 후 URL에서 확인 가능
Step 01

GitHub 레포 만들기

배포할 문서들을 저장할 GitHub 레포를 생성합니다.

  • github.com/new 접속
  • Repository name: docs
  • Visibility: Public
  • Add README: On (초기 파일 필요)
  • Create repository 클릭
💡

이름을 docs로 하면 URL이 eunnwk.github.io/docs로 깔끔하게 나옵니다.

Step 02

GitHub Pages 활성화

  • 레포 상단 Settings 탭 클릭
  • 왼쪽 메뉴 → Pages
  • Source → Deploy from a branch
  • Branch → main / (root) → Save
✅

저장 후 https://eunnwk.github.io/docs URL이 활성화됩니다. 첫 배포 전까지는 404가 뜨는 게 정상입니다.

Step 03

맥북에 클론 & deploy.sh 생성

레포 클론

bash 
cd ~/Desktop
git clone https://github.com/eunnwk/docs
cd docs

deploy.sh 생성

nano 에디터로 파일을 만듭니다:

bash 
nano deploy.sh

에디터가 열리면 아래 내용 붙여넣기:

deploy.sh 
#!/bin/bash
echo "미리보기를 열게요..."
open "$1"

echo ""
echo "배포할까요? (y/n)"
read answer

if [ "$answer" = "y" ]; then
  git add .
  git commit -m "docs: $(date '+%Y-%m-%d %H:%M')"
  git push origin main
  echo "완료! https://eunnwk.github.io/docs"
else
  echo "취소"
fi
⚠️

저장: Ctrl+O → Enter → Ctrl+X

실행 권한 부여

bash 
chmod +x deploy.sh

# 확인 (x 권한 보이면 성공)
ls -la deploy.sh
# → -rwxr-xr-x ...
Step 04

배포 테스트

간단한 테스트 파일로 전체 플로우를 확인합니다.

bash 
./deploy.sh index.html
  • 브라우저에서 미리보기가 열립니다
  • 터미널로 돌아와 y 입력 → 배포 진행
  • 약 1~2분 후 https://eunnwk.github.io/docs 확인
Step 05

실제 사용 방법

세팅 완료 후 실제로 쓰는 방법은 아주 단순합니다.

Claude Code에서

"UX Talk 보조 페이지 HTML 만들어서 ~/Desktop/docs에 저장하고 deploy.sh 실행해줘"

수동으로

Claude가 만든 HTML 파일을 ~/Desktop/docs에 넣고 → ./deploy.sh 파일명.html

index.html을 목차로 관리하기

index.html은 목차 페이지로 사용합니다. 새 문서를 배포할 때마다 index.html에 링크를 추가하는 방식으로 관리합니다. Claude에게 "index.html에 이 문서 링크 추가해줘"라고 하면 자동으로 처리됩니다.

🎯

이 가이드 자체도 ./deploy.sh deploy-guide.html 로 배포되었습니다.

Tips

트러블슈팅

터미널에 bquote> 가 뜰 때

Ctrl+C 로 탈출. heredoc(<< 'EOF') 방식으로 스크립트를 만들 때 백틱이 섞이면 발생합니다. nano 에디터로 직접 만드는 방법을 권장합니다.

Pages URL이 404일 때

Settings → Pages에서 Source 설정이 main / (root)인지 확인. 첫 push 후 1~2분 기다리면 됩니다.

push 시 인증 오류가 날 때

GitHub Personal Access Token이 필요할 수 있습니다. github.com → Settings → Developer Settings → Personal access tokens에서 생성하세요.
