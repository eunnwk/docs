포스팅 자동화 항목

스크립트 실행 시 아래 항목이 자동 처리됩니다.

  • 자동 제목, 본문 마크다운 → HTML 변환
  • 자동 카테고리 생성 및 지정 (호텔 / 카페 / 레스토랑 / 디저트)
  • 자동 해시태그 → 워드프레스 태그 지정
  • 자동 이탈리아2026 태그 자동 추가
  • 자동 대표사진 업로드 및 지정 (images/ 폴더 기준)
  • 자동 Schema.org JSON-LD 구조화 데이터 삽입 (SEO + AI 크롤링)
  • 자동 영문 요약(English Summary) 박스 삽입
  • 자동 쿠팡 파트너스 배너 + 안내 문구 삽입
  • 자동 댓글 비활성화, 핑백 비활성화
  • 수동 대표사진 파일 images/ 폴더에 저장
  • 수동 .md 파일 작성 및 영문 요약 섹션 작성
  • 수동 wp-admin에서 draft 확인 후 publish

스크립트 실행 방법

환경 준비 (최초 1회)

python3 -m venv ~/wp_venv
source ~/wp_venv/bin/activate
pip install requests

매번 실행 전

source ~/wp_venv/bin/activate
cd "폴더경로/outputs"

실행 명령어

# 테스트: 01번 파일 하나만 draft로 업로드
python3 wp_post.py

# 전체 파일 draft로 업로드
python3 wp_post.py --all

# 바로 publish (draft 없이 직접 발행)
python3 wp_post.py --publish

# 전체 파일 바로 publish
python3 wp_post.py --all --publish

새 여행 시리즈 추가 시

  1. 새 폴더 생성 (예: 파리2027_리뷰포스팅/)
  2. wp_post.py 상단 두 줄 수정
  3. .md 파일 작성 후 실행
# wp_post.py 상단에서 수정
POST_DIR    = Path(__file__).parent / "파리2027_리뷰포스팅"
DEFAULT_TAG = "파리2027"

실행 전 가상환경 활성화 필수

가상환경을 활성화하지 않으면 requests 라이브러리를 찾지 못해 에러가 납니다. 터미널을 새로 열 때마다 실행해야 합니다.

source ~/wp_venv/bin/activate

중복 포스팅 주의

스크립트는 이미 올린 포스트가 있는지 확인하지 않습니다. 실수로 두 번 실행하면 동일한 포스트가 두 개 올라갑니다. wp-admin에서 수동으로 삭제해야 합니다.

항상 --all 실행 전에 draft 목록을 wp-admin에서 확인하는 습관을 들이세요.

wp_post.py 위치 변경 시

현재 POST_DIR이 스크립트 기준 상대경로로 설정되어 있습니다. 파일을 다른 폴더로 옮기면 포스팅 폴더를 찾지 못합니다. 옮길 때는 스크립트 상단의 경로를 절대경로로 수정하세요.

# 변경 전 (상대경로)
POST_DIR = Path(__file__).parent / "이탈리아2026_리뷰포스팅"

# 변경 후 (절대경로 예시)
POST_DIR = Path("/Users/ux4all/Documents/이탈리아2026_리뷰포스팅")

Application Password 보안

wp_post.py 파일 안에 워드프레스 비밀번호가 평문으로 저장되어 있습니다. 아래 상황에서 노출되지 않도록 주의하세요.

  • 깃허브 public 저장소에 올리지 않기
  • 다른 사람과 파일 공유 시 비밀번호 줄 삭제 후 공유
  • 비밀번호가 노출된 경우 wp-admin → Profile → Application Passwords에서 즉시 삭제 후 재발급
